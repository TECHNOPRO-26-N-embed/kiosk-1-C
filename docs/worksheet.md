### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

#### 機能1: （機能名）

```
1.スタートメニューを表示
「1:支払い開始」、「2:管理者画面」を表示
2.ユーザーの入力を受け取る
3.入力に応じて分岐する
　-1の場合:支払い処理を呼び起こす
  -2の場合:管理者画面へ移動する
```

#### 機能2: （ポイントカードの有無の確認）

```
1.ポイントカード確認メニュを表示する
「1:無」「2:有」「3購入を取り消す」を表示
2.ユーザーの入力を受け取る
3.「1:無」なら支払選択処理へ移動
4.「有」ならポイントカード読み込み処理へ移動
5.「3購入を取り消す」ならスタート画面に戻る
```

#### 機能3: （支払い選択）

```
1.支払い選択画面を表示
「1:現金」「2:クレジット」「3:ポイント」「4:前画面へ」「5:スタート画面へ」を表示
2.ユーザーの入力を受け取る
3.「1:現金」なら支払画面へ移動
4.「2:クレジット」ならクレジットカード読み取り処理へ移動
5.「3:ポイント」ならカード読み取り画面へ移動
6.「4.前画面へ」なら「ポイントカード有無の確認」画面へ移動
7.「5.スタート画面へ」ならスタート画面へ移動
```

#### 機能4: （年齢確認）

```
1.年齢確認画面を表示
2.「あなたは未成年ですか？」と表示する
「1:はい」「2:いいえ」「3:スタート画面へ」を表示
2.ユーザーの入力を受け取る
3.「1:はい」なら商品読み込み画面へ移動
4.「2:いいえ」なら「この商品はお売りできません」と表示
5.「3:スタート画面へ」ならスタート画面へ移動
```

#### 機能5: （従業員バーコードの確認）

```
1.確認画面を表示
2.従業員バーコードを入力
3.従業員バーコードなら管理者画面へ移動
4.従業員バーコードでないなら、「従業員バーコードを読み込ませてください」と表示
```

#### 機能6: （読み取り処理）

```
1.商品読み取り画面を表示する
　「1:カテゴリ選択へ移動」「2:スタート画面へ」を表示
2.「1:カテゴリ選択へ移動」なら商品読み込み画面へ移動
3.「2:スタート画面へ」ならスタート画面へ移動
```

#### 機能7: （決済確認処理）

```
1.決済確認画面を表示する
2.使用金額を入力を行ってもらう。
3.選択肢を表示する
　「1:支払いを行う」「2:前画面へ」「3:スタート画面へ」を表示
4.「1:支払いを行う」なら決済処理を行う。
5.「2:前画面へ」なら商品読み取り処理へ戻る
6.「2:スタート画面へ」ならスタート画面へ移動
```

#### 機能8: （購入履歴等の出力処理）

```
1.購入履歴等を表示する
2.[0:スタート画面へ]を入力するとスタート画面へ移動

```
